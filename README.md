# 智能部门值班表排班系统

这是一个用于部门值班排班的本地工具。它可以读取人员名单和学生课表 PDF，判断每个人在指定周次、星期和班次是否有空，并辅助完成值班表制作、导出和统计。

软件使用 Streamlit 提供网页界面，但运行在本机电脑上。普通用户使用打包后的版本时，不需要安装 Python，也不需要输入命令。

## 主要功能

1. 人员名单导入

   支持导入 Excel 或 CSV 人员名单。名单中可以记录姓名、班级、部门、职务和校区等信息。

2. 课表批量解析

   支持一次上传多份 PDF 课表，也支持选择课表文件夹或填写本机课表文件夹路径。系统会根据 PDF 文件名识别对应学生姓名，并从课表中提取上课时间。

3. 空闲人员查询

   可以按周次、星期、班次查询哪些人可以值班。系统会自动排除当前时间段有课的人。

4. 排班制作

   可以在页面中选择需要排班的周次，然后按星期和班次为每个位置选择值班人员。下拉框中只显示当前时间段可用的人，减少排班冲突。

5. 排班规则提示

   系统会结合课表和校区判断人员是否可值班。例如南校区通勤风险会在可用人员筛选时自动生效。排班下拉框会显示成员的职务、年级和部门，便于操作人自行判断。

6. Excel 导出和导入

   完成排班后可以导出值班表 Excel。也可以把之前导出的排班表重新导入，继续修改或合并。

7. 值班记录统计

   系统会统计每个人的值班次数，并展示详细记录，方便查看工作量分布。

8. 本地自动保存

   导入的人员名单、已解析课表和值班记录会自动保存在本机，下次打开软件会继续使用上次的数据。

## 普通用户使用打包版

打包后的 Windows 版本通常是一个 zip 压缩包，例如：

```text
DutyScheduler-Windows-日期.zip
```

使用步骤如下：

1. 解压 zip 文件。
2. 打开解压后的 `DutyScheduler` 文件夹。
3. 双击 `DutyScheduler.exe`。
4. 稍等几秒，软件会自动打开浏览器页面。
5. 使用过程中不要关闭黑色命令窗口。关闭该窗口会停止软件服务。
6. 不要只复制 `DutyScheduler.exe` 给别人使用，必须保留它旁边的 `_internal` 文件夹。

如果浏览器没有自动打开，可以手动查看黑色命令窗口中显示的网址，并复制到浏览器地址栏打开。

## 开发环境运行

如果你需要在源码环境中运行项目，请先安装依赖：

```bash
pip install -r requirements.txt
```

然后在项目根目录运行：

```bash
streamlit run duty_app.py
```

如果使用项目自带的虚拟环境，可以运行：

```powershell
.\.venv_pack\Scripts\python.exe -m streamlit run duty_app.py
```

## 数据准备

### 人员名单

人员名单建议使用 Excel 文件。推荐使用更详细的中文表头：

```text
序号
姓名
部门
职务
专业班级
年级
校区
```

对应含义如下：

```text
姓名：成员姓名
部门：成员所在部门，用于后续按部门筛选
职务：干事、干部、部长、副部长、主任团等
专业班级：成员所在专业班级
年级：大一、大二、大三等
校区：旗山北校区、旗山南校区等
```

也兼容英文表头：

```text
Name
Class
Department
Role
Campus
```

校区可以填写南校区、北校区、旗山南校区、旗山北校区等常见写法。系统会尽量自动归一化识别。

### 课表 PDF

课表文件必须是 PDF。文件名中需要包含学生姓名，例如：

```text
张三(2025-2026-1)课表.pdf
李四-课表.pdf
```

如果人员名单中有某位同学，但没有导入对应课表，系统会提示该同学缺少课表。缺少课表的人默认不参与可用人员筛选，避免误排。

## 使用流程

### 第一步：数据导入

1. 打开“数据导入”页面。
2. 上传人员名单 Excel 或 CSV。
3. 上传课表 PDF。可以选择多个 PDF，也可以选择课表文件夹。
4. 等待系统解析完成。
5. 查看页面底部的数据管理区域，确认人员、课表和排班记录数量是否正确。

### 第二步：空闲查询

1. 打开“空闲查询”页面。
2. 选择周次、星期和班次。
3. 根据需要选择部门、职务或校区等筛选条件。
4. 点击查询，查看当前时间段可值班人员。

### 第三步：排班制作

1. 在左侧全局设置中选择需要排班的周次。
2. 打开“排班制作”页面。
3. 选择正在制作的周次。
4. 在每个星期和班次的下拉框中选择值班人员。
5. 点击“保存当前周排班”。
6. 如需继续制作其他周次，在左侧选择对应周次后重复以上步骤。

### 第四步：导出和导入

完成排班后，可以在“排班制作”页面导出 Excel 值班表。

系统支持直接保存到本机文件夹，默认保存位置是项目或软件目录下的 `值班表` 文件夹。也可以在页面中填写其他本机文件夹路径，再点击保存。

如果之后需要继续修改，可以上传之前导出的 Excel，并使用导入功能把旧排班合并回系统。

### 第五步：记录统计

打开“记录统计”页面，可以查看每个人的值班次数排行和详细值班记录。该页面适合在排班完成后检查工作量是否均衡。

## 数据保存位置

软件会自动把数据保存到项目或软件目录下：

```text
data\app_state.json
```

如果使用打包版，数据会保存在解压后的 `DutyScheduler` 文件夹内。移动整个文件夹时，数据也会一起移动。

如果需要重新开始，可以在“数据导入”页面底部的“当前数据管理”区域清空人员名单、课表、排班记录或全部数据。

## 重新打包

如果需要生成新的 Windows 可运行版本，请在项目根目录运行：

## 常见问题

### 为什么某个人在下拉框里找不到？

通常有三种原因：

1. 该同学在这个时间段有课。
2. 该同学缺少课表，系统为了避免误排，把他视为不可值班。
3. 当前班次触发了通勤规则。

可以先到“空闲查询”页面选择相同周次、星期和班次，检查系统筛选结果。

### 为什么上传课表后显示找不到对应人员？

请检查 PDF 文件名是否包含人员名单中的姓名。系统主要通过文件名匹配姓名。

### 为什么别人电脑上不能只运行 exe？

这个软件依赖 `_internal` 文件夹中的运行环境和第三方库。必须把整个 `DutyScheduler` 文件夹一起保留，或者直接发送 zip 发布包。

### 如何清空旧数据？

打开“数据导入”页面，在底部“当前数据管理”区域选择清空人员名单、清空课表、清空排班记录或清空所有数据。

### 打不开软件怎么办？

可以先查看软件旁边的命令窗口是否有错误信息。打包版的错误日志通常会写入：

```text
%LOCALAPPDATA%\DutyScheduler\DutyScheduler.log
```

把日志内容发给开发者，通常可以更快定位问题。
